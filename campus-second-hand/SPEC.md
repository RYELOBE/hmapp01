# 校园二手交易平台 - 系统规格说明书

## 1. 项目概述

### 1.1 项目背景
随着大学生消费升级，每年产生大量闲置物品。校园二手交易市场潜力巨大，但现有平台缺乏校园场景适配。本项目旨在构建一个功能完善、体验优良的校园二手交易平台。

### 1.2 项目目标
- 为大学生提供便捷的二手物品交易平台
- 实现完整的校园身份认证体系
- 集成AI智能问答服务
- 提供用户端和管理端双端系统
- 支持完整的交易流程

## 2. 技术架构

### 2.1 技术栈
- **前端框架**: Vue 3 + JavaScript
- **UI组件库**: Arco Design Pro
- **后端框架**: Spring Boot 3.x (Java 17)
- **数据库**: MySQL 8.0
- **缓存**: Redis 7.0
- **AI服务**: 通义千问API / OpenAI API
- **构建工具**: Maven + Vite
- **部署**: Docker

### 2.2 系统架构
```
┌─────────────────────────────────────────┐
│              用户端 (Vue3)               │
│   首页 | 商品浏览 | 个人中心 | AI助手    │
├─────────────────────────────────────────┤
│              运营端 (Vue3)               │
│   用户管理 | 商品审核 | 订单管理 | 统计  │
├─────────────────────────────────────────┤
│              API网关 (Spring Boot)       │
│   用户服务 | 商品服务 | 订单服务 | AI服务 │
├─────────────────────────────────────────┤
│              数据存储层                   │
│   MySQL | Redis | 文件存储              │
└─────────────────────────────────────────┘
```

## 5. API接口设计

### 5.1 RESTful API规范
- 统一前缀: `/api/v1`
- 认证方式: JWT Token
- 响应格式: JSON
- 状态码: 200成功, 400参数错误, 401未认证, 403无权限, 500服务器错误

### 5.2 核心接口

#### 认证模块
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/logout` - 用户登出

#### 用户模块
- `GET /api/v1/user/profile` - 获取用户信息
- `PUT /api/v1/user/profile` - 更新用户信息
- `POST /api/v1/user/verify` - 校园认证

#### 商品模块
- `GET /api/v1/product/list` - 商品列表
- `GET /api/v1/product/{id}` - 商品详情
- `POST /api/v1/product` - 发布商品
- `PUT /api/v1/product/{id}` - 更新商品
- `DELETE /api/v1/product/{id}` - 删除商品

#### 订单模块
- `POST /api/v1/order` - 创建订单
- `GET /api/v1/order/list` - 订单列表
- `PUT /api/v1/order/{id}/pay` - 支付订单
- `PUT /api/v1/order/{id}/confirm` - 确认收货

#### AI模块
- `POST /api/v1/ai/chat` - AI对话
- `GET /api/v1/ai/recommend` - 智能推荐

### 7.1 认证授权
- JWT Token认证
- Token有效期24小时
- Refresh Token 7天

### 7.2 数据安全
- 密码加密存储 (BCrypt)
- 敏感信息脱敏
- SQL注入防护
- XSS防护

### 7.3 接口安全
- 请求频率限制
- 参数校验
- 跨域配置

## 8. 性能要求

### 8.1 响应时间
- 页面加载 < 3秒
- API响应 < 500ms

### 8.2 并发能力
- 支持1000+并发用户
- 支持500+商品同时在线

### 8.3 可用性
- 系统可用性 > 99.9%
- 数据备份每日一次
