# 2026-04-30 工作日志

## Bug 审查与修复

全面审查了校园二手平台优化后的代码，发现并修复了以下问题：

### 🔴 严重 Bug（已修复）
1. **SQL 注入风险** — `ItemRepository` 和 `OrderRepository` 的 `findByPage/countByFilter/findVendorsWithStats/findBuyersWithStats` 等方法中，keyword/status/category 参数直接拼接到 SQL 字符串。**全部改为 PreparedStatement 参数化查询**。
2. **OrderController.confirm() 参数不匹配** — 路径定义为 `/{id}/confirm` 但用了 `@RequestBody Map` 取值，前端也传 body。**改为 `@PathVariable("id") Long orderId`**。

### 🟠 中等问题（已修复）
3. **listMyItemsPaged() 性能缺陷** — 先全表查 findByPage 再内存 filter sellerId，totalCount 是全表 count 而非当前卖家的。**新增 findBySellerIdPaged SQL 层分页 + countBySellerIdWithFilter 精确计数**。

### 🟡 轻微问题（已修复）
4-5. **Portal/Ops Layout 图标引用错误** — navLinks/menuConfig/statCards 用字符串形式的 icon 名称（如 `"icon-home"`）传给 `<component :is>`，但 ArcoDesign 图标未全局注册会导致渲染空白。**全部改为 emoji 方案**（如 `iconEmoji: "🏠"`）。
6. **ItemService.listItemsPaged 中 showMine 分页** 也存在全量查再内存截断的问题，同步修复为 SQL 层分页。

### 涉及文件（第一轮）
- 后端：ItemRepository.java, OrderRepository.java, ItemService.java, OrderController.java
- 前端：portal/Layout.vue, ops/Layout.vue, ops/dashboard/index.vue, portal/router/index.js

## 第二轮 Bug 审查（继续深入）

### 🔴 严重 Bug（已修复）
7. **OpsReviewController.queue() 返回格式不兼容前端** — 前端 `reviews/index.vue` 期望分页结构 `{ items, totalCount, pageNo, pageSize }`，后端只返回 `{ items: [...] }` 且无分页参数。**改为支持 status/pageNo/pageSize 参数，调用 ReviewService.getReviewQueuePaged() 返回标准分页格式。**
8. **OrderService.getMinePaged() 全量加载+内存分页 OOM 风险** — 先 `findByBuyerId/SellerId` 查全部订单再 subList 截断。**在 OrderRepository 新增 findByBuyerIdPaged/findBySellerIdPaged/countByBuyerIdWithFilter/countBySellerIdWithFilter，SQL 层完成分页和筛选。**

### 🟠 中等 Bug（已修复）
9. **UploadService 无文件类型/大小校验** — 可上传任意文件（含可执行文件），无上限。**增加 Content-Type 白名单（jpeg/png/gif/webp/bmp）+ 5MB 大小限制。**
10. **createItem imageUrls 类型不匹配** — 后端 record 定义 `String imageUrls`，前端 PublishItem 传 `string[]` 数组。**改为 `Object imageUrls` + ItemService 内部兼容 List 和 String 两种类型。**

### 🟡 轻微 Bug（已修复）
11. **OpsReviewController approve/reject 返回格式不一致** — 返回 `{ message: "ok" }` 而其他接口返回 `{ code: 200, message: ... }`。**统一为 code 200 格式。**
12. **Portal 多页面使用未注册的 Arco 图标组件** — `<icon-image>`, `<icon-check-circle>`, `<icon-plus>`, `<icon-location>`, `<icon-shopping-cart>`, `<icon-search>` 未从 @arco-design/web-vue 显式导入，渲染空白。**全部替换为 emoji（📷✅➕📍🛒🔍）。**
