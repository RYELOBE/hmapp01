# 校园二手交易平台产品业务逻辑文档

## 一、产品概述

### 1.1 产品定位
校园二手交易平台是面向高校师生设计的在线二手物品交易系统，致力于为校园用户提供便捷、安全、可信赖的二手物品发布与购买服务。

### 1.2 目标用户
- **买家用户**：在校学生，希望购买性价比高的二手物品
- **卖家用户**：在校学生，希望出售闲置物品
- **未登录用户**：访客，可浏览商品但无法购买或发布
- **运营人员**：平台管理员，负责商品审核、用户管理、内容管理

### 1.3 核心业务流程
平台围绕商品生命周期展开，包括商品发布→审核→上架→交易→完成的完整闭环。

---

## 二、用户角色与权限

### 2.1 用户角色定义

| 角色 | 权限描述 |
|------|----------|
| 未登录用户 | 浏览商品列表、查看商品详情、搜索商品 |
| 买家 | 购买商品、收藏商品、提交订单、发布评价 |
| 卖家 | 发布商品、管理商品、处理订单、查看销售统计 |
| 运营人员 | 审核商品、审核评价、审核圈子内容、管理用户、查看数据统计 |

### 2.2 角色交互规则
- 用户可同时拥有买家和卖家角色
- 角色切换通过用户中心进行
- 运营账号与普通用户账号完全隔离

---

## 三、商品发布与审核流程

### 3.1 商品发布流程
```
卖家登录 → 进入发布页面 → 填写商品信息 → 上传图片 → 提交审核
                                              ↓
                                         保存草稿（可选）
```

### 3.2 商品信息字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 60字符以内 |
| 分类 | 下拉选择 | 是 | 预设分类树 |
| 成色 | 下拉选择 | 是 | 全新/99新/95新/9成新/8成新/7成新 |
| 价格 | 数字 | 是 | 正数，最多两位小数 |
| 原价 | 数字 | 否 | 用于展示折扣 |
| 数量 | 数字 | 是 | 大于0 |
| 图片 | 图片 | 是 | 最多9张，第一张为主图 |
| 描述 | 富文本 | 否 | 详细描述商品信息 |

### 3.3 审核流程
```
卖家提交 → 待审核状态 → 运营审核
                          ↓
              ┌──────────┴──────────┐
              ↓                     ↓
           审核通过              审核拒绝
              ↓                     ↓
         商品上架              通知卖家修改
```

### 3.4 审核规则
- 审核时限：24小时内完成
- 拒绝原因必填
- 拒绝后可重新提交审核

---

## 四、评价发布与审批机制

### 4.1 评价触发条件
- 订单完成后15天内可评价
- 仅买家可评价卖家
- 每个订单仅可评价一次

### 4.2 评价内容
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 评分 | 1-5星 | 是 | 综合评分 |
| 内容 | 富文本 | 是 | 评价内容，支持格式设置 |
| 图片 | 图片 | 否 | 最多5张 |

### 4.3 评价审核流程
```
买家提交评价 → 待审核状态 → 运营审核
                              ↓
                   ┌──────────┴──────────┐
                   ↓                     ↓
                审核通过              审核拒绝
                   ↓                     ↓
              展示在商品详情页       评价不展示，通知买家
```

### 4.4 评价展示规则
- 仅审核通过的评价展示
- 按时间倒序排列
- 显示卖家回复

---

## 五、收藏功能实现逻辑

### 5.1 收藏操作
- 用户点击收藏按钮添加收藏
- 收藏成功后按钮状态变为"已收藏"
- 收藏计数实时更新

### 5.2 收藏列表
- 展示用户收藏的所有商品
- 支持按收藏时间排序
- 支持批量取消收藏

### 5.3 收藏状态同步
- 登录状态下收藏数据同步到服务端
- 未登录状态下收藏数据保存在本地存储
- 登录后自动合并本地收藏

## 六、消息中心设计

### 6.1 消息类型
| 类型 | 触发场景 | 示例 |
|------|----------|------|
| 系统通知 | 系统公告、活动通知 | 平台升级维护通知 |
| 交易消息 | 订单状态变更 | 卖家已发货 |
| 审核消息 | 审核结果通知 | 商品审核通过 |
| 互动消息 | 评价回复、私信 | 卖家回复了您的评价 |

### 6.2 消息状态
- **未读**：新消息，未查看
- **已读**：已查看但未操作
- **已处理**：已完成相关操作

### 6.3 消息交互
- 顶部导航显示未读消息数量
- 点击消息图标展开消息列表
- 支持批量标记为已读

## 七、校园圈子内容管理流程

### 7.1 圈子内容类型
- **图文帖**：文字+图片内容
- **纯文字帖**：仅文字内容

### 7.2 内容发布流程
```
用户登录 → 进入圈子页面 → 点击发布 → 编辑内容 → 提交审核
```

### 7.3 内容字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 50字符以内 |
| 内容 | 富文本 | 是 | 支持格式设置和图片 |
| 话题标签 | 标签 | 否 | 最多5个 |

### 7.4 内容审核流程
```
用户提交 → 待审核状态 → 运营审核
                          ↓
              ┌──────────┴──────────┐
              ↓                     ↓
           审核通过              审核拒绝
              ↓                     ↓
         展示在圈子首页          通知用户修改
```

### 7.5 圈子互动
- 支持点赞、评论
- 评论无需审核，实时显示
- 支持分享到其他平台

## 八、订单交易流程

### 8.1 订单状态流转
```
创建订单 → 待付款 → 已付款 → 已发货 → 已完成
              ↓              ↓
           取消订单       申请退款
```

### 8.2 订单状态说明
| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待付款 | 订单已创建，等待付款 | 付款、取消 |
| 已付款 | 付款成功，等待发货 | 申请退款 |
| 已发货 | 卖家已发货，等待收货 | 确认收货、申请退款 |
| 已完成 | 买家确认收货 | 评价商品 |
| 已取消 | 订单已取消 | 重新购买 |
| 已退款 | 退款已完成 | 无 |

---

## 九、数据流转规则

### 9.1 核心数据流
```
用户操作 → API请求 → 服务端处理 → 数据库更新 → 返回响应 → 前端更新
```

### 9.2 数据同步机制
- 用户登录状态：Token共享，自动刷新
- 业务数据：服务端拉取，确保一致性
- 缓存策略：热门数据多级缓存

### 9.3 并发处理
- 商品库存：乐观锁机制
- 订单创建：库存预占
- 高并发限流

## 十一、业务规则汇总

### 11.1 用户规则
- 用户名3-20字符，唯一
- 密码至少6位
- 角色变更需审核

### 11.2 商品规则
- 标题≤60字符
- 价格≤99999元
- 图片最多9张，格式jpg/png/webp

### 11.3 评价规则
- 订单完成后15天内可评价
- 评价内容需审核
- 支持图片上传（最多5张）

### 11.4 圈子规则
- 标题≤50字符
- 内容需审核
- 禁止发布违规内容
