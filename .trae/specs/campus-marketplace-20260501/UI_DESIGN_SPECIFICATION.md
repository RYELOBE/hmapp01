# 校园二手交易平台UI设计优化规范说明书

## 一、项目背景与目标

### 1.1 现状分析

当前校园二手交易平台已实现基础功能框架，采用微前端架构（Shell + Portal + OPS），后端基于Spring Boot，前端使用Vue 3 + Vite技术栈。现有UI组件库为Arco Design Vue，但整体视觉呈现存在以下改进空间：

**视觉层面的不足**

当前设计缺乏统一的设计语言体系，各页面风格不协调。首页轮播区域仅使用渐变背景块，缺少真实图片支撑，视觉冲击力不足。商品卡片虽然集成了基础信息展示，但整体布局较为平淡，未能有效突出商品特色。色彩体系虽有蓝色主调，但应用不够系统化，部分页面存在配色混乱问题。

**交互体验的局限**

现有交互设计较为基础，缺少微动效和状态过渡动画。用户操作反馈不够明显，部分按钮和链接缺乏hover和active状态的视觉区分。加载状态的展示方式单一，主要依赖简单的loading动画。空状态和错误状态的页面设计不够友好，未能提供清晰的引导信息。

**参考设计的差距**

对比目标参考站点（湾数通流通利用平台）的设计风格，现有平台在以下方面存在差距：整体视觉层次不够清晰，信息密度与留白的平衡欠佳；导航结构与信息架构的呈现方式较为简单；数据可视化组件的运用不够充分；响应式布局的适配精度有待提升。

### 1.2 优化目标

本次UI优化的核心目标是打造一套统一、专业、现代化的视觉体系，在保持平台功能完整性的同时，显著提升用户体验和视觉品质。具体而言，优化后的平台应达到以下标准：建立清晰的设计语言体系，确保所有页面的视觉一致性；通过精细的配色方案和排版设计，提升整体美感和可读性；增加合理的动效和微交互，增强用户操作的反馈感和愉悦度；优化信息架构和布局结构，提升用户获取信息的效率；确保跨设备、跨浏览器的显示一致性。

## 二、设计语言体系

### 2.1 色彩系统

**主色调（Primary Colors）**

品牌主色采用蓝色系，体现科技感、专业性和信任感。主蓝色设定为#165DFF，这是Arco Design的默认蓝色，视觉明亮且具有较高的辨识度。在不同场景下，主蓝色的应用方式有所区别：纯蓝色背景（#165DFF）用于主要按钮、链接文字、选中状态和重点强调元素；浅蓝色背景（#E6F1FF）用于标签背景、hover状态填充和次级强调区域；深蓝色（#0E42D2）用于按钮的hover状态和深色强调场景。

**辅助色彩（Secondary Colors）**

辅助色体系的构建遵循Arco Design的色彩规范，结合平台特性进行适配。成功色（Success）使用#00B42A，主要应用于审核通过、订单完成、收藏成功等正向操作反馈；警告色（Warning）使用#FF7D00，用于待处理提醒、价格异常等需要关注的场景；危险色（Danger）使用#F53F3F，适用于删除操作、审核拒绝、退款等负面操作；信息色（Info）使用#14C9C9，用于信息提示和次级引导。

**中性色彩（Neutral Colors）**

中性色用于页面背景、卡片、边框和文字等元素。一级文字色（#1D2129）用于标题和重要信息；二级文字色（#4E5969）用于正文和次要信息；三级文字色（#86909C）用于辅助说明和占位符；四级文字色（#C9CDD4）用于禁用状态和分割线；边框色（#E5E6EB）用于卡片边框和分隔线；背景色（#F2F3F5）用于页面背景和hover状态；纯白背景（#FFFFFF）用于主要内容和卡片。

**渐变色彩（Gradient Colors）**

在首页轮播和重点区域使用渐变色增强视觉吸引力。品牌渐变（Brand Gradient）从#165DFF渐变至#4080FF，适用于首页横幅和重点引导区域；成功渐变（Success Gradient）从#00B42A渐变至#23C343，适用于促销信息和成功提示；科技渐变（Tech Gradient）从#165DFF经#0E42D2至#0B2E8C，适用于深色主题区域和数据面板。

### 2.2 字体系统

**字体家族（Font Family）**

平台整体采用系统字体栈，确保在不同操作系统上的良好显示。中文主要使用"-apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif"字体栈；数字和英文使用"'SF Pro Display', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif"；代码和特殊场景使用"'JetBrains Mono', 'Fira Code', 'Consolas', monospace"。

**字体层级（Font Hierarchy）**

通过字号、字重和颜色的组合建立清晰的字体层级体系。一级标题（H1）使用28px字号和700字重，颜色为一级文字色（#1D2129），行高1.3；二级标题（H2）使用22px字号和600字重，颜色为一级文字色，行高1.4；三级标题（H3）使用18px字号和600字重，颜色为二级文字色（#4E5969），行高1.5；正文内容使用14px字号和400字重，颜色为二级文字色，行高1.6；辅助说明使用12px字号和400字重，颜色为三级文字色（#86909C），行高1.5。

### 2.3 间距系统

**基础间距单位（Spacing Unit）**

采用4px作为基础间距单位，所有间距值均为4的倍数。xs级间距为4px，用于紧密相关元素之间的间距；sm级间距为8px，用于同类元素组内的间距；md级间距为16px，用于不同元素组之间的间距；lg级间距为24px，用于页面主要区块之间的间距；xl级间距为32px，用于页面级的大区块分隔；xxl级间距为48px，用于页面顶部和底部的留白区域。

**组件内间距（Component Padding）**

不同类型组件采用不同的内边距标准。卡片组件（Card）采用24px的内边距；按钮组件（Button）的水平内边距为20px，垂直内边距为8px至12px；表单项（Form Item）采用16px的内边距；表格单元格（Table Cell）采用12px至16px的内边距；列表项（List Item）的内边距为12px至16px。

### 2.4 圆角系统

**组件圆角（Border Radius）**

根据组件类型和重要程度应用不同的圆角值。大型卡片和模态框使用12px的圆角；中型组件如按钮、输入框使用6px的圆角；小型元素如标签、徽章使用4px的圆角；圆形元素如头像使用50%的圆角（完全圆形）；图片容器使用8px的圆角。

### 2.5 阴影系统

**层级阴影（Elevation Shadows）**

建立三级阴影体系表示不同的层级关系。一级阴影（sm）用于卡片默认状态，参数为"0 1px 2px 0 rgba(0, 0, 0, 0.05)"；二级阴影（md）用于卡片hover状态和悬浮元素，参数为"0 4px 8px 0 rgba(0, 0, 0, 0.08)"；三级阴影（lg）用于弹出层和下拉菜单，参数为"0 8px 16px 0 rgba(0, 0, 0, 0.12)"。

### 2.6 动效规范

**过渡时长（Transition Duration）**

根据交互类型确定合适的过渡时长。快速交互（hover、active）使用150ms的过渡时长；标准过渡（展开、收起）使用250ms的过渡时长；复杂动画（页面切换、模态弹出）使用350ms的过渡时长。

**缓动函数（Easing Functions）**

不同场景使用不同的缓动函数。标准缓动（ease-out）用于元素进入视图；标准缓动（ease-in）用于元素离开视图；弹性缓动（ease-in-out）用于元素的尺寸变化；弹性效果（cubic-bezier(0.5, 0, 0, 1)）用于按钮点击反馈。

## 三、页面设计方案

### 3.1 首页设计方案

**布局架构（Layout Architecture）**

首页采用单栏布局结构，从上至下依次为：顶部导航栏（固定）、搜索区域、分类导航、轮播横幅、商品网格、底部信息栏。整体宽度最大限制为1400px，居中显示，左右留白区域使用浅灰色背景填充。

**顶部导航栏（Top Navigation）**

导航栏高度设定为64px，背景色为纯白色，底部使用1px的浅灰色边框。左侧放置平台Logo和"校园集市"文字标识；中部为搜索输入框，宽度400px，带有搜索图标和"请搜索商品"占位提示；右侧为用户操作区域，包含登录注册按钮或用户头像下拉菜单，以及购物车图标和消息图标。导航栏在滚动时保持固定，添加轻微的阴影效果。

**搜索区域（Search Section）**

搜索框采用圆角设计，高度48px，宽度占满导航栏中部。搜索框左侧显示分类下拉选择器，提供"全部分类"和各商品分类选项；搜索按钮使用主蓝色背景（#165DFF），包含搜索图标和"搜索"文字。搜索区域背景使用微渐变效果，从纯白至浅灰。

**分类导航（Category Navigation）**

分类导航采用横向滚动标签栏设计，高度48px，背景为纯白色，使用8px的垂直外边距与上下内容区隔。分类标签样式：未选中状态为灰色文字和透明背景，选中状态为主蓝色文字和浅蓝色背景（#E6F1FF），hover状态为浅灰色背景。标签采用胶囊形状，左右各16px的内边距，4px的圆角。

**轮播横幅（Banner Carousel）**

轮播区域高度为280px，采用圆角设计（12px），外层容器带有阴影效果。轮播图采用真实商品场景图片，图片上叠加半透明渐变遮罩（从透明至深色），底部显示图片指示器（dots）和左右箭头。指示器样式为小型圆形，未选中状态为半透明白色，选中状态为纯白色。自动播放间隔设定为5秒，切换动画使用淡入淡出效果。

**商品网格（Product Grid）**

商品网格采用响应式布局：桌面端（>1200px）每行6个商品；笔记本端（992px-1200px）每行4个商品；平板端（768px-992px）每行3个商品；手机端（<768px）每行2个商品。商品卡片之间的间距为16px。网格上方显示商品统计信息"共XXX件好物"和排序选项。加载状态使用骨架屏动画展示。

**商品卡片（Product Card）**

商品卡片采用白色背景，8px的圆角边框，hover状态下向上移动4px并增加阴影强度。卡片结构从上至下为：商品图片（宽高比1:1，圆角8px）、商品标题（最多2行显示，超出省略）、价格信息（原价的删除线显示）、卖家信息和收藏按钮。图片区域添加浅灰色占位背景，加载时显示loading动画。底部显示卖家头像（24px圆形）、卖家昵称和发布时间。

**底部信息栏（Footer）**

底部信息栏包含平台版权信息、联系方式和帮助链接。背景使用浅灰色（#F2F3F5），内边距为32px。文字使用三级文字色（#86909C），链接使用主蓝色。

### 3.2 商品详情页设计方案

**页面布局（Page Layout）**

商品详情页采用左右分栏布局：左侧60%宽度用于商品图片和详情信息展示；右侧40%宽度用于购买操作区域，宽度固定320px。整体最大宽度1400px，居中显示。

**左侧信息区（Left Info Area）**

图片展示区占据顶部空间。图片画廊采用主图加缩略图列的布局，主图区域高度480px，支持点击放大查看；缩略图列显示在主图下方，每行5张，高度60px，支持点击切换主图。图片支持手势缩放和左右滑动切换。图片下方为商品基本信息区域，包含标题、价格、成色、分类、浏览量、收藏数等信息。

商品描述区域位于基本信息下方，使用卡片样式展示。标题为"商品详情"，内容为卖家填写的富文本描述，支持图片显示。卖家信息卡片位于描述下方，展示卖家头像、昵称、好评率、注册时间、累计售出数量等，点击卖家头像可进入卖家店铺页面。

**右侧操作区（Right Action Area）**

购买操作区采用固定定位，始终显示在视口右侧。顶部显示当前价格和原价对比；中部为SKU选择区域（数量选择器，最小1，最大为商品库存）；底部为操作按钮区，包含"立即购买"（主按钮，全宽）和"加入购物车"（次按钮，全宽）两个主要操作；按钮下方为收藏按钮和分享按钮。

AI问答入口显示在操作区底部，以悬浮按钮形式存在。按钮样式为圆形，直径48px，主蓝色背景，白色图标，hover状态下放大至56px并增加阴影。点击后展开AI对话抽屉。

### 3.3 买家中心设计方案

**页面布局（Page Layout）**

买家中心采用左侧导航加右侧内容的布局结构。左侧导航栏宽度240px，固定定位；右侧内容区宽度自适应，最小宽度800px。顶部显示当前角色切换标签（买家/卖家）。

**左侧导航（Side Navigation）**

导航菜单采用垂直列表布局。菜单项包含图标和文字，选中状态使用主蓝色背景（#E6F1FF）和主蓝色文字。菜单分组使用灰色分隔线区分，各分组标题使用小号灰色文字。主要菜单项包括：我的订单、我的收藏、购物车、收货地址、评价管理、账户设置。

**订单列表（Order List）**

订单列表页顶部包含订单状态筛选标签栏和搜索框。筛选标签包括全部、待付款、待发货、待收货、已完成、已取消、退款中，全部高亮显示。订单卡片采用时间倒序排列，每张订单卡片包含订单编号、创建时间、卖家信息、商品列表、订单金额、订单状态和操作按钮。

订单卡片内的商品列表采用横向排列，支持点击展开查看更多。操作按钮根据订单状态动态显示：待付款状态显示"去支付"和"取消订单"；已付款状态显示"提醒发货"；已发货状态显示"确认收货"和"申请退款"；已完成状态显示"评价商品"和"再次购买"；退款中状态显示"查看退款进度"。

**我的收藏（My Favorites）**

收藏列表采用网格布局，每行4个商品卡片（桌面端）。卡片样式与首页商品卡片一致，但增加"取消收藏"按钮。页面顶部显示收藏统计和批量操作区域，支持全选和批量取消收藏。

**购物车（Shopping Cart）**

购物车页面采用表格式布局，列分别为选择框、商品信息、单价、数量、金额、操作。商品信息列包含商品图片（80px方形）、商品标题和卖家信息。数量选择使用Arco的InputNumber组件，支持直接输入和加减按钮操作。底部固定显示结算栏，包含已选商品统计、总金额和结算按钮。

### 3.4 卖家中心设计方案

**页面布局（Page Layout）**

卖家中心与买家中心共享相同的布局结构，仅左侧导航菜单项不同。卖家中心导航包括：商品管理、发布商品、订单管理、销售统计、卖家设置。

**商品管理（Product Management）**

商品管理列表顶部包含筛选区域，包括状态筛选（下拉多选）、分类筛选、关键词搜索和高级筛选按钮。列表采用表格式展示，列为：商品图片、商品标题、商品价格、库存、审核状态、上架状态、创建时间、操作。

审核状态使用彩色标签显示：待审核（黄色#FFB026）、审核通过（绿色#00B42A）、审核拒绝（红色#F53F3F）。审核拒绝状态需要显示拒绝原因，点击可展开查看。操作列包含编辑（编辑商品信息）、上下架（切换上架状态）、删除（需二次确认）按钮。

**发布商品（Publish Product）**

商品发布采用分步表单设计，共三个步骤：基本信息、图片上传、详细描述。每一步都有进度指示器显示当前步骤。表单验证采用即时验证方式，错误信息显示在对应字段下方。

第一步"基本信息"包含：商品标题（必填，最多60字符）、商品分类（必填，下拉选择）、商品成色（必填，单选：全新、几乎全新、良好、一般）、商品价格（必填，数字输入，最小0.01）、商品原价（选填，用于展示折扣）、商品数量（必填，最小1）。第二步"图片上传"要求至少上传1张，最多9张图片，第一张为主图。上传区域支持拖拽上传，预览区域显示已上传图片，支持调整主图和删除操作。第三步"详细描述"使用富文本编辑器，支持文字格式和图片插入，提供模板参考。

### 3.5 运营后台设计方案

**整体风格（Overall Style）**

运营后台采用深色侧边栏加浅色内容区的经典后台管理布局。侧边栏宽度260px，使用深灰色背景（#1D2129）；内容区使用浅灰色背景（#F2F3F5），页面卡片使用白色背景。

**侧边导航（Side Navigation）**

侧边栏顶部显示平台Logo和管理员头像。导航菜单分为两级：一级菜单为功能分组标题，使用较小字号和灰色文字；二级菜单为具体功能项，包含图标和文字，选中状态使用主蓝色背景高亮显示。菜单项包括：工作台、商品审核、订单管理、供方管理、需方管理、用户管理、角色管理、页面配置、系统设置。

**工作台（Dashboard）**

工作台页面顶部为欢迎语和日期时间显示。核心数据展示区采用四个统计卡片横排布局，分别为：待审核商品、今日通过、今日驳回、商品总数。统计卡片采用图标加数字的布局，图标使用彩色圆形背景。

快捷操作区采用2行4列的卡片网格布局，每个卡片包含图标、标题和描述文字。快捷操作包括：待审核商品、订单监控、供方管理、需方管理、用户管理、页面配置、最近活动、帮助文档。

最近活动列表采用时间线样式展示，显示最新的系统事件。事件类型包括用户注册、商品审核、订单生成、异常告警等，每种类型使用不同的图标和颜色标识。

**商品审核（Product Review）**

审核列表页采用筛选栏加表格的布局。筛选栏包含审核状态（下拉选择：全部、待审核、已通过、已拒绝）、分类筛选、关键词搜索和时间范围选择。表格列为：商品图片（缩略图）、商品标题、商品价格、卖家信息、提交时间、审核状态、操作。

审核详情页采用分栏布局：左侧显示商品完整信息，包括商品图片（可放大查看）、商品标题、价格、描述、卖家信息；右侧为审核操作区，包含审核意见填写框、"审核通过"和"审核拒绝"两个操作按钮。拒绝操作必须填写拒绝原因，提供常用拒绝原因的快捷选择。

**数据可视化（Data Visualization）**

运营后台的关键数据展示区域使用图表组件进行可视化呈现。使用Arco Design的图表组件或ECharts展示数据趋势。折线图用于展示用户增长、商品发布量、订单量等时间序列数据；饼图用于展示商品分类分布、用户角色分布等占比数据；柱状图用于展示日/周/月统计数据对比。

### 3.6 登录注册页面设计方案

**整体布局（Overall Layout）**

登录注册页面采用左右分栏布局：左侧60%为品牌展示区，右侧40%为表单操作区。品牌展示区使用品牌渐变背景（#165DFF至#4080FF），展示平台Logo、标语和核心功能介绍。表单区为白色背景，居中放置登录或注册表单卡片。

**登录表单（Login Form）**

登录表单卡片宽度400px，居中显示。表单包含用户名输入框、密码输入框（带显示/隐藏切换）和登录按钮。表单下方提供"记住密码"复选框、"忘记密码"链接和"注册账号"链接。登录按钮为主按钮样式，占满表单宽度。

**注册表单（Register Form）**

注册表单卡片同样宽度400px。表单流程为三步引导式：第一步选择注册身份，提供"买家"和"卖家"两个大按钮选择，可多选；第二步填写基本信息，包括用户名（必填）、密码（必填，最少6位）、确认密码（必填，需与密码一致）、手机号（选填）；第三步验证并完成，显示注册成功提示和跳转登录按钮。

## 四、组件设计规范

### 4.1 按钮组件（Button）

**按钮类型（Button Types）**

主要按钮（Primary Button）使用主蓝色背景（#165DFF）和白色文字，hover状态下背景加深（#0E42D2），active状态下背景更深（#0B2E8C），disabled状态下使用浅灰背景和灰色文字。

次要按钮（Secondary Button）使用白色背景和主蓝色边框（1px），主蓝色文字，hover状态下背景变为浅蓝色（#E6F1FF），active状态下背景更深（#C6D9F1）。

文字按钮（Text Button）不使用背景和边框，仅使用主蓝色文字，hover状态下显示下划线。

危险按钮（Danger Button）使用红色背景（#F53F3F）和白色文字，hover状态下背景加深，用于删除等危险操作。

**按钮尺寸（Button Sizes）**

小按钮（Small）高度32px，水平内边距16px，文字14px，用于表格内操作。

中等按钮（Medium）高度36px，水平内边距20px，文字14px，为默认尺寸，用于大多数场景。

大按钮（Large）高度44px，水平内边距24px，文字16px，用于主要操作如登录、提交。

### 4.2 卡片组件（Card）

**卡片样式（Card Styles）**

标准卡片使用白色背景、8px圆角边框、轻微阴影（sm级）。卡片头部（Header）使用16px的内边距，底部有1px的浅灰分隔线（可选）；卡片内容区（Content）使用20px的内边距；卡片底部（Footer）使用16px的内边距，顶部有1px的浅灰分隔线（可选）。

**卡片状态（Card States）**

默认状态使用基础阴影和白色背景。悬浮状态（hover）向上移动4px并使用更深的阴影（md级），过渡时长150ms。选中状态使用主蓝色边框（2px），背景变为浅蓝色（#E6F1FF）。禁用状态使用灰色调，opacity降低至0.6，cursor变为not-allowed。

### 4.3 表格组件（Table）

**表格样式（Table Styles）**

表头使用浅灰色背景（#F7F8FA），文字使用二级文字色（#4E5969），字号13px，字重600。表头单元格内边距为12px水平、14px垂直。表格行高设定为52px以确保足够的点击区域。行悬浮状态使用浅灰背景（#F7F8FA）。

**表格交互（Table Interactions）**

行选择使用左侧复选框，选中行使用浅蓝背景（#E6F1FF）。行悬浮使用轻微的背景色变化。排序图标显示在列标题右侧，支持升序和降序切换。列宽可调整，鼠标移动到列边框时显示调整光标。

### 4.4 表单组件（Form）

**输入框样式（Input Styles）**

输入框高度40px，内边距左右16px、上下10px。边框使用1px浅灰色（#E5E6EB），圆角6px。聚焦状态使用主蓝色边框和浅蓝色阴影（0 0 0 3px rgba(22, 93, 255, 0.1)）。错误状态使用红色边框和红色阴影，错误提示文字显示在输入框下方。

**表单布局（Form Layout）**

垂直布局（Vertical Layout）每个表单项占一行，标签在输入框上方，左对齐。水平布局（Horizontal Layout）标签和输入框在同一行，标签宽度固定120px，输入框占满剩余空间。行内布局（Inline Layout）多个表单项在同一行，用于筛选条件区域。

### 4.5 标签组件（Tag）

**标签样式（Tag Styles）**

默认标签使用灰色背景（#F2F3F5）和灰色文字（#4E5969）。主要标签使用浅蓝背景（#E6F1FF）和主蓝色文字。成功标签使用浅绿背景（#E8FFEA）和绿色文字（#00B42A）。警告标签使用浅橙背景（#FFF7E6）和橙色文字（#FF7D00）。危险标签使用浅红背景（#FFECE8）和红色文字（#F53F3F）。

标签可关闭，使用小号关闭图标，hover时显示。标签圆角为4px，内边距为8px水平、4px垂直。

### 4.6 状态组件（Status）

**加载状态（Loading）**

骨架屏（Skeleton）用于内容加载，使用灰色渐变动画从左至右扫过。旋转加载（Spin）使用Arco默认的加载动画，可指定大小（小16px、中24px、大32px）。按钮内加载显示在按钮文字左侧。

**空状态（Empty）**

空状态插图使用简约的SVG图形，配色使用灰色系。空状态文字使用二级文字色，说明性文字使用三级文字色。操作按钮（如果有）使用次要按钮样式。

**错误状态（Error）**

错误状态使用红色系配色。错误图标使用实心圆加感叹号的图形。错误提示文字使用红色。提供"重试"按钮供用户操作。

## 五、响应式设计规范

### 5.1 断点定义（Breakpoints）

移动端（Mobile）宽度小于768px，布局调整为单栏，导航变为底部标签栏。平板端（Tablet）宽度768px至992px，布局调整为双栏，导航保持侧边栏但可折叠。笔记本端（Laptop）宽度992px至1200px，标准布局，导航保持侧边栏。桌面端（Desktop）宽度大于1200px，最大内容宽度1400px。

### 5.2 各端适配策略

**移动端适配（Mobile Adaptation）**

侧边导航栏收起为汉堡菜单，点击展开为全屏抽屉。商品网格从每行6个缩减为每行2个。商品详情页从左右分栏变为上下单栏。操作按钮组从并排变为垂直堆叠。表格变为卡片列表形式。

**平板端适配（Tablet Adaptation）**

侧边导航栏可折叠/展开，点击图标仅显示图标，再次点击展开完整菜单。商品网格调整为每行3至4个。详情页保持左右布局但比例调整为55:45。

**桌面端适配（Desktop Adaptation）**

完整的侧边导航栏展示。标准的多栏商品网格。完整的数据可视化图表。并排的操作按钮组。

## 六、无障碍设计规范

### 6.1 色彩对比度

文字与背景的色彩对比度应符合WCAG 2.1 AA级标准：正文文字（14px以下）与背景的对比度至少4.5:1；大号文字（18px以上或14px粗体）与背景的对比度至少3:1；交互元素（按钮、链接）与背景的对比度至少3:1。

### 6.2 键盘可访问性

所有交互元素可通过Tab键聚焦，聚焦状态使用明显的轮廓指示（主蓝色2px轮廓）。下拉菜单、模态框等可使用方向键导航。模态框打开时应捕获键盘焦点，关闭后焦点应返回到触发元素。

### 6.3 屏幕阅读器支持

所有图片提供有意义的alt属性。表单输入提供关联的label标签。按钮和链接使用描述性文字。表格提供表头关联。

## 七、业务逻辑完整性验证清单

### 7.1 核心功能模块验证

**用户认证模块**

用户注册流程需验证：表单验证规则正确执行，包括用户名唯一性检查、密码强度验证、确认密码一致性校验。角色选择功能正常，支持买家、卖家单选或双选。注册成功后自动登录并跳转至对应页面。

用户登录流程需验证：用户名密码正确时登录成功并跳转；错误时显示明确的错误提示；记住密码功能正确记录登录状态；登录过期后自动跳转登录页。

**商品管理模块**

商品发布流程需验证：必填字段未填写时阻止提交；图片上传张数限制（1-9张）正确执行；价格格式验证正确；成色选项单选功能正常；草稿保存功能正常；提交审核后状态正确更新。

商品审核流程需验证：待审核列表正确加载；审核通过操作正确更新商品状态；审核拒绝操作正确更新状态并记录原因；拒绝原因正确通知卖家。

**订单管理模块**

订单创建流程需验证：库存不足时阻止下单；收货地址未填写时提示完善；订单金额计算正确；订单号生成唯一。

订单状态流转需验证：待付款状态下可取消；已付款状态下可申请退款；已发货状态下可确认收货；状态变更记录完整。

### 7.2 权限控制验证

**角色权限隔离**

买家无法访问卖家功能（发布商品、订单发货）；卖家无法访问其他卖家商品的管理；运营人员无法访问买家中心的购买功能；超级管理员权限完整覆盖运营后台所有功能。

**数据范围隔离**

用户只能查看和操作自己的订单；卖家只能管理自己发布的商品；运营人员可查看所有数据但操作权限受限。

## 八、UI优化实施计划

### 8.1 第一阶段：基础组件优化

本阶段聚焦于全局样式变量定义和基础组件的样式规范化。具体工作包括：定义完整的CSS变量体系并应用到全局样式表；规范化按钮、输入框、卡片等基础组件的样式；建立统一的图标使用规范；优化表单验证提示样式。

### 8.2 第二阶段：核心页面重构

本阶段对用户访问量最大的核心页面进行UI重构。工作内容包括：首页轮播图区域重新设计，增加真实图片支持；商品卡片组件重新设计，优化信息布局和交互细节；商品详情页整体重构，采用新的左右分栏布局；搜索结果页面的筛选和排序交互优化。

### 8.3 第三阶段：功能页面优化

本阶段优化各功能模块页面的用户体验。重点工作包括：买家中心的订单列表和收藏页面重新设计；卖家中心的商品管理和发布页面优化；购物车页面的交互优化；登录注册页面的整体改版。

### 8.4 第四阶段：运营后台优化

本阶段提升运营后台的视觉一致性和工作效率。工作内容包括：统一运营后台的设计风格与用户端保持一致；工作台数据可视化图表的设计和实现；审核流程页面的交互优化；侧边导航菜单的优化。

### 8.5 第五阶段：动效和细节打磨

本阶段添加微交互动效并进行细节打磨。工作内容包括：页面切换动画的实现；hover和active状态的动效添加；加载状态、骨架屏、空状态的统一设计；响应式布局的细节优化。

## 九、验收标准

### 9.1 视觉验收

所有页面符合设计规范定义的颜色、字体、间距标准。响应式布局在各个断点下表现正常。所有组件状态（默认、hover、active、disabled、loading）样式正确。动画过渡流畅，无明显卡顿。

### 9.2 功能验收

所有业务流程按照产品需求正常执行。用户权限控制正确执行。数据操作符合业务规则校验。接口错误和异常情况正确处理。

### 9.3 性能验收

页面首次加载时间不超过3秒（网络正常情况下）。主要交互操作响应时间不超过200ms。动画帧率保持在50fps以上。无明显的布局偏移（CLS）和交互延迟（INP）。
