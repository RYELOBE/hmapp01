# 校园二手交易平台 - 完整页面框架与功能修复 Spec

## Why
当前首页UI升级只实现了Hero区域内容，但缺少关键的**页面框架结构**（顶部导航栏+布局容器），导致：
1. 页面显示不完整，没有参考图片中的顶部导航栏
2. 点击其他页面无响应（路由/权限问题）
3. 整体用户体验与设计稿差距较大

## What Changes
- **重构PortalLayout为完整页面框架**：添加顶部导航栏（参考第一张图样式）
- **修复路由跳转问题**：确保所有导航链接可点击且正常工作
- **完善页面布局层级**：Header → Hero → Content 的正确嵌套关系
- **保持蓝白色调一致性**：导航栏也采用蓝白配色方案

## Impact
- Affected specs: campus-marketplace-ui-upgrade-20260506 (延续)
- Affected code:
  - `packages/apps/campus-app/src/views/portal/Layout.vue` (主要改造 - 添加导航栏)
  - `packages/apps/campus-app/src/views/portal/home/index.vue` (微调 - 适配新布局)
  - `packages/apps/campus-app/src/router/index.js` (检查路由配置)

## ADDED Requirements

### Requirement: 完整的顶部导航栏
系统 SHALL 在门户页面提供固定的顶部导航栏，包含：
- **左侧**：Logo "校园二手平台" + 主导航菜单（首页、全部商品、卖家精选、校园圈子、客服中心）
- **右侧**：搜索图标 + 通知图标 + 用户头像下拉菜单（登录/注册、个人中心、退出）
- **样式**：白底/浅蓝底，高度64px，固定定位(sticky/fixed)，阴影效果
- **响应式**：移动端折叠为汉堡菜单

#### Scenario: 用户浏览不同页面
- **WHEN** 用户点击导航栏中的任意链接
- **THEN** 页面平滑跳转到对应路由，URL正确更新
- **AND** 当前页面导航项高亮显示

### Requirement: 正确的页面布局层级
系统 SHALL 确保页面结构符合以下层级：
```
┌─────────────────────────────────────┐
│         Header (导航栏)              │ ← 固定顶部
├─────────────────────────────────────┤
│                                     │
│          Hero 区域                  │ ← 全屏宽度
│      (标题+搜索+CTA按钮)            │
│                                     │
├─────────────────────────────────────┤
│       分类入口 / 商品列表           │ ← 内容区域
│                                     │
└─────────────────────────────────────┘
```

#### Scenario: 滚动页面时
- **WHEN** 用户向下滚动页面
- **THEN** 导航栏始终可见（sticky定位）
- **AND** Hero区域随滚动移出视口

## MODIFIED Requirements

### Requirement: PortalLayout组件
**原需求**：简单的router-view容器
**修改为**：包含完整导航栏的页面框架组件
**原因**：用户期望看到完整的页面布局，而非孤立的内容区
**影响范围**：所有portal子页面都会继承此导航栏

### Requirement: 路由交互逻辑
**原需求**：基本路由跳转
**修改为**：增强的路由导航体验
- 未登录状态：显示"登录/注册"按钮，点击跳转登录页
- 已登录状态：显示用户头像+下拉菜单（个人中心、我的订单、退出）
- 所有链接都有hover效果和过渡动画

## REMOVED Requirements
无

```
┌─[🔵 校园二手平台]──[首页]─[全部商品]─[卖家精选]─[校园圈子]─[客服中心]─────[🔍][🔔][👤 张三 ▼]─┐

│   高度：64px                                                                                    │
```
