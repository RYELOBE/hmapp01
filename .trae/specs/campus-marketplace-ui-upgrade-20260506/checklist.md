# 校园二手交易平台 - 首页UI升级 - 验证清单

## 功能验证

- [ ] 首页无需登录即可正常访问
  - **验证方法**：清除浏览器缓存和Cookie，直接访问 `/portal/home`
  - **预期结果**：页面完整显示，不跳转到登录页
  - **优先级**：P0

- [ ] Hero区域全屏显示且视觉冲击力强
  - **验证方法**：查看首页首屏内容
  - **预期结果**：
    - Hero区域占据首屏主要空间（高度≥500px）
    - 蓝白渐变背景或高质量背景图
    - 大标题"校园二手交易平台"清晰醒目
    - 副标题说明平台价值
    - CTA按钮（立即发布/去逛逛）位置显眼
  - **优先级**：P0

- [ ] 搜索框集成在Hero区域内且易用
  - **验证方法**：在Hero区域的搜索框输入关键词并搜索
  - **预期结果**：
    - 搜索框尺寸大（高度≥44px）、圆角设计
    - 占位符文案清晰（如"搜索你想要的宝贝..."）
    - 支持回车搜索和点击搜索按钮
    - 搜索后商品列表正确更新
  - **优先级**：P0

- [ ] 分类快捷入口功能正常
  - **验证方法**：点击不同的分类图标
  - **预期结果**：
    - 分类图标使用SVG或高质量图标（非emoji）
    - 布局为网格状（4-5列）
    - 点击分类后商品列表正确过滤
    - 当前选中分类有高亮状态
    - 悬停有交互反馈效果
  - **优先级**：P1

- [ ] 商品展示网格布局合理
  - **验证方法**：浏览商品列表区域
  - **预期结果**：
    - 响应式网格布局（桌面5列、平板3-4列、手机2列）
    - 商品卡片包含：图片、标题、价格、卖家信息
    - 价格显示突出（颜色/字号）
    - 卡片悬停有上浮+阴影加深动效
    - "加载更多"按钮样式现代化
  - **优先级**：P1

- [ ] 排序功能正常工作
  - **验证方法**：切换"最新"、"价格升"、"价格降"
  - **预期结果**：
    - 排序选项UI清晰
    - 切换后商品列表顺序正确变化
    - 无页面刷新，体验流畅
  - **优先级**：P1

## 视觉设计验证

- [ ] 蓲白色调统一且协调
  - **验证方法**：整体审视页面各区域配色
  - **预期结果**：
    - 主色调为蓝色系（#165DFF, #4080FF等）
    - 背景以白色和浅灰蓝为主
    - 文字颜色层次分明（深灰主文字、中灰次要文字）
    - 强调色（按钮、链接）使用品牌蓝
    - 无突兀杂色出现
  - **优先级**：P0

- [ ] 排版与间距符合现代设计规范
  - **验证方法**：检查各模块的字体大小、行距、间距
  - **预期结果**：
    - 标题字号层级清晰（Hero标题≥36px，区块标题≥24px）
    - 正文字号舒适（14-16px）
    - 模块间距一致（20px-40px）
    - 内容区域最大宽度限制（1200px-1400px居中）
    - 留白充足，不拥挤
  - **优先级**：P1

- [ ] 圆角、阴影等细节处理精致
  - **验证方法**：观察卡片、按钮、输入框等元素
  - **预期结果**：
    - 圆角统一（按钮8-12px，卡片12-16px）
    - 阴影柔和自然（非生硬黑边框）
    - 按钮有渐变或纯色填充
    - 边框线细腻（1px浅色边框或无边框）
  - **优先级**：P1

- [ ] 图标风格统一且高质量
  - **验证方法**：检查所有图标（分类、按钮、装饰性图标）
  - **预期结果**：
    - 使用线性或面性图标（统一风格）
    - 图标尺寸适中（24px-48px）
    - 颜色与主题协调
    - 无模糊或像素化问题
  - **优先级**：P1

## 交互体验验证

- [ ] 动画流畅且增强体验
  - **验证方法**：触发各种交互场景
  - **预期结果**：
    - Hero区域入场动画（淡入/上浮，时长300-500ms）
    - 商品卡片悬停上浮（transform: translateY(-4px)）
    - 分类图标悬停缩放或变色
    - 按钮点击有按压反馈
    - 页面切换过渡动画平滑
    - 动画不影响性能（60fps）
  - **优先级**：P2

- [ ] 响应式布局适配良好
  - **验证方法**：使用浏览器DevTools切换不同设备尺寸
  - **预期结果**：
    - **桌面端 (≥1280px)**：5列商品网格，Hero全宽显示
    - **平板端 (768px-1279px)**：3-4列网格，搜索框自适应
    - **移动端 (<768px)**：2列网格，Hero高度缩小，单列布局
    - 无横向滚动条
    - 文字大小随屏幕调整
    - 触摸目标足够大（≥44px）
  - **优先级**：P1

- [ ] 空状态和加载状态友好
  - **验证方法**：
    - 搜索不存在的关键词
    - 等待数据加载时观察状态
  - **预期结果**：
    - 空状态有友好提示（图标+文案+"换个关键词试试"）
    - 加载状态有Spinner或Skeleton占位
    - 错误状态有重试按钮
  - **优先级**：P1

## 性能验证

- [ ] 首页加载性能达标
  - **验证方法**：使用Lighthouse或DevTools Performance面板
  - **预期结果**：
    - 首次内容绘制(FCP) < 2秒
    - 最大内容绘制(LCP) < 3秒
    - 累积布局偏移(CLS) < 0.1
    - 总阻塞时间(TBT) < 200ms
  - **优先级**：P1

- [ ] 图片资源优化
  - **验证方法**：检查Network面板图片加载情况
  - **预期结果**：
    - Hero背景图使用WebP格式或适当压缩
    - 商品图片懒加载实现
    - 图片尺寸适配不同设备（srcset或CSS media query）
    - 无过大未压缩图片（单张<500KB）
  - **优先级**：P2
